5. State machine suggestion
idle -> listening -> thinking -> speaking -> idle

6. Production guidance
- Use speaking only during actual TTS playback
- Use listening only when waiting for user voice input
- Use thinking for backend/model/tool latency
- Keep idle calm and always present
